Mac+Github+Hexo+Next搭建个人博客

Posted on 2019-04-07

前言

一直以来都有写文章的习惯，不管是收集资料还是发表个人见解，最开始把文章都放在有道笔记，但是有道笔记有个不好的点就是不便于分享，后来搭建了GitBook，但是里面的界面不太美观。后来接触到了CSDN,但是太多的广告。后来接触到了GitHub-Pages+Hexo，不需要服务器，而且免费。

为什么要用Hexo?

Hexo是一个快速、简洁且高效的博客框架，可以在极短的时间一键生成所有的静态页面，支持 Markdown，丰富的插件，一键部署到GitHub Pages, Heroku 或其他网站。

搭建环境准备

Node安装

Node.js是国外的网站，下载的服务可能不太友好，可以去国内的Nodejs中文官网下载，选择适合自己的版本安装就好了，安装过程只需要下一步下一步。网址：http://nodejs.cn/download/

检查安装是否成功

node -v 
npm -v

Git安装

Git是一个版本控制工具，主要是在Hexo发布起作用。将本地的Hexo博客（静态文件）同步至Github上。

网址：https://git-scm.com/downloads

检验是否安装成功：

git —version

博客本地安装

安装hexo

通过npm来安装hexo

npm install -g hexo-cli

创建网站

hexo init <folder>
cd <folder>

例如：

hexo init blog
cd blog

网站目录介绍

参数 描述 默认值
source_dir 资源文件夹，这个文件夹用来存放内容。 source
public_dir 公共文件夹，这个文件夹用于存放生成的站点文件。 public
tag_dir 标签文件夹 tags
archive_dir 归档文件夹 archives
category_dir 分类文件夹 categories
code_dir Include code 文件夹 downloads/code
i18n_dir 国际化（i18n）文件夹 :lang
skip_render 跳过指定文件的渲染，您可使用 glob 表达式来匹配路径。
